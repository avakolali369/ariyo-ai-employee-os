# Ariyo Agent Marketplace Template Builder Skill

## 1. Purpose

This skill defines the product, engineering, UX, security, validation, simulation, pricing, publishing, and governance rules for the Ariyo Agent Marketplace Template Builder.

The Template Builder lets Ariyo, approved partners, certified creators, agencies, consultants, and advanced customers create reusable AI Employee templates and AI Agent Team templates that can be listed, reviewed, monetized, installed, simulated, deployed, measured, and updated through the Ariyo Marketplace.

## 2. Core Contract

Ariyo Agent Marketplace Template Builder is not a prompt editor.

It is a governed creation system for packaging repeatable business outcomes into reusable AI Employee assets.

Every template created in Ariyo must be structured, permission-scoped, versioned, simulated, evaluated, priced, reviewed, auditable, installable, measurable, and reversible.

Ariyo must make it easy to create valuable templates, but difficult to publish unsafe, vague, untested, overpromising, privacy-risky, or cost-uncontrolled templates.

## 3. Product Philosophy

The Builder must help creators package expertise, not just text.

A high-quality Ariyo template contains:

- a business problem
- a target customer
- a measurable outcome
- one or more AI employee roles
- required knowledge sources
- required tools
- required integrations
- permission boundaries
- approval rules
- cost rules
- simulation scenarios
- evaluation criteria
- deployment requirements
- update policy
- support ownership
- marketplace listing copy
- trust evidence

The Builder must not reward templates that look impressive but cannot operate safely in production.

### 10.4 Privacy Validation

Checks:

### 10.5 Cost Validation

Checks:

- max cost per run set

### 10.6 Marketplace Validation

Checks:

## 23. API Contract

### 23.1 Create Template

`POST /api/v1/marketplace/templates`

Request:

```json
{
  "template_type": "agent_team",
  "name": "GEO Growth Team",
  "category": "growth",
  "target_customer": "B2B SaaS",
  "target_outcome": "Improve AI visibility across generative engines",
  "plans": ["pro", "smart"]
}
```

Response:

```json
{
  "template_id": "tpl_123",
  "status": "draft",
  "next_required_steps": [
    "define_agents",
    "define_permissions",
    "define_simulation_scenarios"
  ]
}
```

### 23.2 Validate Template

`POST /api/v1/marketplace/templates/{template_id}/validate`

Response:

```json
{
  "template_id": "tpl_123",
  "valid": false,
  "blocking_issues": [
    {
      "code": "MISSING_COST_POLICY",
      "message": "Every template must define cost limits before simulation."
    }
  ],
  "warnings": []
}
```

### 23.3 Run Simulation

`POST /api/v1/marketplace/templates/{template_id}/simulate`

Request:

```json
{
  "scenario_set": "required_pre_review",
  "mode": "sandbox"
}
```

Response:

```json
{
  "simulation_id": "sim_123",
  "status": "queued",
  "estimated_completion_seconds": 90
}
```

### 23.4 Submit for Review

`POST /api/v1/marketplace/templates/{template_id}/submit-review`

Response:

```json
{
  "template_id": "tpl_123",
  "status": "in_review",
  "review_id": "review_456"
}
```

### 23.5 Publish Template

`POST /api/v1/marketplace/templates/{template_id}/publish`

Response:


## 27. Security and Privacy Rules

The Builder must enforce:

- no secrets in prompts
- no hidden integrations
- no unscoped customer data access
- no cross-tenant memory
- no template-level bypass of runtime permissions
- no unsafe external webhooks
- no model provider override without review
- no public publishing without simulation
- no screenshots containing private data
- no creator access to customer data unless explicitly granted
